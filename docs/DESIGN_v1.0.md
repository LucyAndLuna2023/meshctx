# meshctx v1.0 — 世界第一Agent系统架构设计

> 设计日期: 2026-05-09
> 设计目标: 全面超越 Hermes / OpenClaw / WorkBuddy / Copaw / Claude Cowork
> 设计原则: 架构 / 性能 / 智能 / 记忆 / 扩展性 / 开源性 六维碾压

---

## 一、竞争对手分析

### 1.1 Hermes (Nous Research)
| 优势 | 劣势 |
|------|------|
| 持久化记忆跨session | 单Agent架构，无多Agent协同 |
| 技能系统(Skills)可复用 | 记忆是扁平KV，无层次结构 |
| 多平台Gateway(飞书/Telegram) | 无自我反思/元认知 |
| session_search全文检索 | 上下文组装是被动的(等用户问) |
| cronjob定时任务 | 无预测性上下文预加载 |

### 1.2 OpenClaw
| 优势 | 劣势 |
|------|------|
| 多消息平台桥接 | 本身不是Agent，只是网关 |
| Webhook事件系统 | 无记忆管理 |
| 轻量部署 | 无智能决策能力 |

### 1.3 WorkBuddy / Copaw
| 优势 | 劣势 |
|------|------|
| 编码任务执行 | 上下文窗口限制严重 |
| 工具调用链 | 无跨会话记忆 |
| | 无多Agent编排 |

### 1.4 Claude Cowork (Anthropic)
| 优势 | 劣势 |
|------|------|
| Claude模型深度集成 | 封闭生态，仅Anthropic |
| 多步推理 | 无持久化记忆(依赖模型上下文) |
| 文件操作 | 无插件生态 |

### 1.5 关键空白 — meshctx的决胜点
1. **无人实现真正的层次记忆** → meshctx做Ebbinghaus遗忘曲线+向量图混合检索
2. **无人有元认知循环** → meshctx做自我反思→自动技能创建
3. **无人在Agent层做预测** → meshctx做预测性上下文预加载
4. **无人真正开源插件生态** → meshctx做MCP原生+插件市场

## 二、v1.0 六维重构设计

### 维度一: 架构 — 事件驱动微内核

```
┌─────────────────────────────────────────────────────────┐
│                    meshctx Kernel                        │
│  ┌─────────┐  ┌──────────┐  ┌──────────┐               │
│  │ Event   │  │ Plugin   │  │ Resource │               │
│  │ Bus     │  │ Manager  │  │ Governor │               │
│  └────┬────┘  └────┬─────┘  └────┬─────┘               │
│       │            │              │                      │
│  ┌────┴────────────┴──────────────┴──────────────────┐  │
│  │                 Plugin Slots                       │  │
│  │  [Memory] [Tools] [Gateway] [Scheduler] [LLM]    │  │
│  │  [Code]   [Files] [Browser] [Vision]  [Audio]    │  │
│  └────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**核心变化**:
- 从单体FastAPI → 微内核+插件架构
- 事件总线(RabbitMQ/Redis Streams)解耦所有模块
- 每个能力都是可热插拔插件
- 资源调控器防止OOM/CPU过载

### 维度二: 性能 — 预测+流式+分层

```
性能三层:
┌────────────────────────────────────┐
│ L1: 预测层 (Pre-fetch)             │
│ - 时间模式学习 (工作日9点→股票)     │
│ - 项目上下文预加载                  │
│ - 用户行为预测模型                  │
├────────────────────────────────────┤
│ L2: 流式层 (Streaming)             │
│ - 上下文流式组装(不等待全量)        │
│ - LLM token streaming + 边生成边执行│
│ - 工具调用并行化(无依赖并发)        │
├────────────────────────────────────┤
│ L3: 缓存层 (Tiered Cache)          │
│ - L1: 进程内存 (热点数据)           │
│ - L2: Redis (跨进程共享)            │
│ - L3: 分布式存储 (S3/MinIO)         │
└────────────────────────────────────┘
```

**关键指标目标**:
- 上下文组装延迟: < 50ms (当前 ~200ms)
- 记忆检索延迟: < 10ms (当前 ~100ms)
- 首次响应时间: < 500ms (端到端)

### 维度三: 智能 — 元认知循环 (Meta-Cognition Loop)

```
每个任务执行后:
┌───────────────────────────────────────────────┐
│           Meta-Cognition Loop                  │
│                                                │
│  1. Self-Evaluate                              │
│     ├─ 任务成功? (布尔)                         │
│     ├─ 质量评分 (0-1)                           │
│     ├─ 耗时分析                                 │
│     └─ 错误分类 (工具/推理/知识/权限)           │
│                                                │
│  2. Pattern Extraction                         │
│     ├─ 成功模式 → 自动创建Skill                 │
│     ├─ 失败模式 → 创建防护规则                   │
│     └─ 新发现 → 更新知识图谱                    │
│                                                │
│  3. Knowledge Graph Update                     │
│     ├─ 实体关系抽取                             │
│     ├─ 项目间知识迁移                           │
│     └─ 遗忘曲线更新(不重要信息衰减)             │
│                                                │
│  4. Behavior Adjustment                        │
│     ├─ 调整工具选择策略                         │
│     ├─ 调整上下文组装权重                       │
│     └─ 更新用户偏好模型                         │
└───────────────────────────────────────────────┘
```

### 维度四: 记忆 — Ebbinghaus遗忘曲线 + 混合检索

```
记忆层次架构:
┌──────────────────────────────────────────────────┐
│ L0: 感知记忆 (Sensory)                            │
│     - 当前对话流                                  │
│     - 容量: 无上限(流式)                          │
│     - 保留: 会话期间                              │
├──────────────────────────────────────────────────┤
│ L1: 工作记忆 (Working Memory)                     │
│     - 当前任务上下文                              │
│     - 容量: ~10K tokens                           │
│     - 保留: 任务期间                              │
│     - 检索: 直接引用                              │
├──────────────────────────────────────────────────┤
│ L2: 短时记忆 (Short-term)                         │
│     - 近7天会话摘要 + 关键决策                    │
│     - 容量: ~100K tokens                          │
│     - 衰减: Ebbinghaus曲线(1h→50%, 1d→30%)      │
│     - 检索: 向量 + 时间衰减加权                   │
├──────────────────────────────────────────────────┤
│ L3: 长时记忆 (Long-term)                          │
│     - 全部历史压缩为语义向量                      │
│     - 容量: 无上限                                │
│     - 检索: 混合检索(向量+图+关键词)              │
│     - 索引: FAISS/Milvus + Neo4j知识图谱         │
├──────────────────────────────────────────────────┤
│ L4: 归档记忆 (Archival)                           │
│     - 跨项目通用知识                              │
│     - 自动去重+合并                               │
│     - 定期整理(weekly compaction)                 │
└──────────────────────────────────────────────────┘

检索策略 (Hybrid Retrieval):
FinalScore = α*VectorSim + β*GraphRelevance + γ*RecencyBias - δ*DecayFactor

α=0.4 (语义相似度)
β=0.3 (知识图谱关联)
γ=0.2 (时间衰减)
δ=0.1 (遗忘惩罚)
```

## 三、多Agent协作架构 (Orchestra Mode)

```
Orchestrator Pattern:
┌────────────────────────────────────────────────────┐
│                 Orchestrator Agent                   │
│  - 接收用户意图                                     │
│  - 分解为子任务                                     │
│  - 分配专业Agent                                    │
│  - 合并结果                                         │
│                                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐         │
│  │ Coder    │  │ Researcher│  │ DevOps   │         │
│  │ Agent    │  │ Agent     │  │ Agent    │         │
│  │          │  │           │  │          │         │
│  │ - 写代码 │  │ - 搜索    │  │ - 部署   │         │
│  │ - 调试   │  │ - 分析    │  │ - 监控   │         │
│  │ - 测试   │  │ - 总结    │  │ - 运维   │         │
│  └──────────┘  └──────────┘  └──────────┘         │
│                                                     │
│  共享: Memory Hub (所有Agent读写同一记忆空间)       │
└────────────────────────────────────────────────────┘

关键能力:
- 任务DAG依赖解析
- Agent间消息传递(通过Memory Hub)
- 冲突检测与解决
- 超时+重试策略
- 结果验证Agent(独立审查)
```
